# Uniqor Cinematic Website Blueprint

## 1. Brand Position

**Company:** Uniqor  
**Core positioning:** The Core Intelligence Stack for Critical Systems.  
**Primary homepage line:** Agent infrastructure for critical systems.  
**Category:** Institutional AI infrastructure / governed agent infrastructure.  
**Audience:** Regulated enterprises, legal institutions, healthcare systems, government agencies, finance, institutional partners, and investors.

Uniqor should not read as a productivity tool, SaaS page, or generic AI assistant. The site positions Uniqor as the trust layer institutions need before AI can scale safely.

## 2. Emotional Target

The visitor should feel that Uniqor is controlled, serious, technically deep, and inevitable.

Desired reaction:

> This is not software. This is the intelligence infrastructure layer for institutions.

For investors:

> This company understands where AI is actually going.

### Section 2 — The Trust Gap

**Eyebrow:** The trust gap  
**Headline:** AI is scaling faster than institutional trust.

**Body:**  
When decisions span systems, policies, and departments, isolated copilots become operational risk. Outputs may be impressive, but without native governance, auditability, and accountability, institutions cannot defend what the AI decided or why it decided it.

**Cards:**

1. **Fragmented context**  
   Legal sees one record. Compliance sees another. Operations works from a third. Intelligence breaks when the institution is represented as disconnected fragments.  
   Signal: Context drift.

2. **Governance after the fact**  
   Validation is often bolted on after output generation. That creates audit gaps exactly where regulated decisions require evidence, ownership, and control.  
   Signal: Audit exposure.

3. **Unstable decision integrity**  
   Model behavior changes across teams, time, workflows, and data freshness. In high-stakes environments, inconsistency becomes systemic risk.  
   Signal: Variance detected.

**Visual:** fragmented enterprise system map.

Motion: disconnected red and amber lines flicker. Nodes show legal, compliance, ops, finance, risk, policy, and data. The center then activates as Uniqor and stabilizes the graph with cyan/teal pathways.

### Section 4 — Audit Trail

**Eyebrow:** Audit-ready intelligence  
**Headline:** A decision trace that survives scrutiny.

**Body:**  
Each output is treated as an institutional decision event. Context, policy, reasoning, validation, output, and ledger entry become a traceable chain rather than an after-the-fact narrative.

**Decision trace:**

1. **Input** — Request context, role, permission, scope.  
   The system captures the decision boundary before reasoning begins.

2. **Context** — Dynamic state retrieval + institutional memory.  
   Authoritative data, prior decisions, and current workflow state are assembled.

3. **Policy check** — SOPs, regulations, playbooks, thresholds.  
   Rules of the institution are applied as part of the inference path.

4. **Reasoning** — Cross-system reconciliation + validation.  
   Contradictions are detected, resolved, or escalated for human authority.

5. **Validation** — Source integrity, confidence, variance control.  
   The output is tested against available evidence and decision constraints.

6. **Output** — Structured recommendation + rationale.  
   Decision owners receive a defensible output, not an opaque answer.

7. **Audit ledger** — Immutable lineage + outcome monitoring.  
   What was decided, when, by whom, and on what basis remains available.

Motion: each stage illuminates progressively. Metadata reveals on hover. A cyan-to-teal-to-amber rail shows the decision moving through validation.

## 6. Motion System

### Global Feel

Motion should be elegant, expensive, controlled, and non-playful. Avoid bounce effects. Use cinematic easing and low-amplitude motion.

### Libraries

- Three.js for hero governed core.
- GSAP for the hero load sequence and precise timeline choreography.
- Framer Motion for section reveal, SVG paths, cards, meters, and pillars.
- Lenis for smooth inertial scrolling.

### Timing Principles

| Motion Type | Duration | Ease |
|---|---:|---|
| Section reveal | 0.75s–0.95s | cubic-bezier(0.16, 1, 0.3, 1) |
| Hero sequence | 2.3s–2.8s total | power3.out |
| SVG path draw | 1.0s–1.8s | cubic-bezier(0.16, 1, 0.3, 1) |
| Core rotation | continuous slow | linear + inertia |
| Hover glow | 180ms–260ms | ease-out |
| Data pulse | 2.2s–3.4s | ease-in-out |

### Reduced Motion

If `prefers-reduced-motion: reduce` is enabled:

- Disable Lenis.
- Render the Three.js core as a static state after first frame.
- Disable repeated UI pulse animations.
- Keep content visible without relying on animation.

## 7. Interaction System

### Buttons

- Soft cyan/teal glow on hover.
- Slight upward movement.
- No elastic bounce.
- Magnetic movement can be added in production only if subtle.

### Cards

- Dark glass panels.
- Animated border scan on hover.
- Metadata reveal on hover for audit cards.
- Light movement should be barely perceptible.

### Graphs and Diagrams

- SVG lines draw progressively.
- Nodes activate in sequence.
- Risk lines use red/amber; governed lines transform into cyan/teal/mint.

### Hero Core

- Slight cursor-reactive rotation.
- Slow continuous rings.
- Particles move along data streams.
- Core pulse increases subtly during scroll transition into the trust gap section.

## 9. Responsive Behavior

### Desktop

- Hero uses two-column editorial layout.
- Three.js core receives majority visual width.
- Architecture visual is sticky.
- Audit trail is horizontal.
- Use cases and governance cards are four-column.

### Tablet

- Two-column grids collapse where needed.
- Audit trail becomes two-column.
- Use cases become two-column.
- Core labels remain visible if space allows.

### Mobile

- Navigation reduces to logo + CTA.
- Hero visual drops below copy.
- Core labels hide to preserve clarity.
- Cards become single-column.
- Audit trail becomes vertical.
- Three.js core height reduces.
